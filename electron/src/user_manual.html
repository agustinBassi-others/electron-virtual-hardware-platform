Manual de Uso
Virtual Hardware Platform

 

Author: Agustin Bassi

 

Introducción

1.1 - Descripción conceptual

 

El propósito de la aplicación es ser una herramienta que permita la enseñanza de programación en Sistemas Embebidos mediante una interfaz amigable y fácil de utilizar.

 

A grandes rasgos, el proyecto se compone de dos partes fundamentales:

  • Una aplicación de escritorio multiplataforma creada en Electron (NodeJS, HTML y CSS) que contiene periféricos virtuales.
  • Una biblioteca para Sistemas Embebidos creada en Lenguaje C para múltiples arquitecturas que permite acceder a los periféricos virtuales de la aplicación mediante el puerto USB-Serie.

 

1.2 - Instalación general

 

Descargar de este link el programa para el sistema operativo deseado (Linux o Windows) y seguir los pasos de instalación convencionales.

 

Una vez instalado abrir el programa y explorar las diferentes solapas.

 

1.3 - Conexión con el sistema embebido

 

Para conectarse, es necesario que el sistema embebido posea puerto USB-Serie.

 

  1. Conectar el sistema embebido PC.
  2. Dirigirse a la solapa periféricos del programa.
  3. Elegir el puerto donde se encuentra conectado el embebido y la velocidad de conexión.
  4. Clickear sobre la llave de conexión.
    1. Si cambió a color verde, la conexión está establecida.
    2. Si quedó en color rojo chequear que el puerto seleccionado sea el correcto.
    3. En caso de haber conectado el sistema embebido después de iniciar el programa, presionar CTRL + R para recargar y seleccionar el puerto y la velocidad correspondientes.

 

1.4 - Ejecutar el “Hola Mundo!”

 

Requerimientos:

  • Tener instalado el programa (1.1).
  • IDE para compilar y descargar el programa al Sistema Embebido (ciaa-ide, embeddedIDE, Arduino Sketch, etc.).

 

Pasos:

  1. Abrir el IDE para compilar/descargar el programa al Sistema Embebido y crear un nuevo proyecto.
  2. Crear tres archivos con los siguientes nombres: main.c, virtual_hardware.h, virtual_hardware.c
  3. Dirigirse a la solapa Código Fuente de Virtual Hardware Platform.
  4. Presionar el botón Copiar al portapapeles virtual_hardware.h y luego, dentro del archivo virtual_hardware.h presionar CTRL + V.
  5. Presionar el botón Copiar al portapapeles virtual_hardware.c y luego, dentro del archivo virtual_hardware.c presionar CTRL + V.
  6. Presionar el botón Copiar al portapapeles ejemplo1.c y luego, dentro del archivo main.c presionar CTRL + V.
  7. Dentro del archivo virtual_hardware.h seleccionar el Sistema Embebido que se va a programar descomentando el correspondiente. Es necesario que haya una única plataforma seleccionada al momento de compilar.
  8. Compilar el programa y descargarlo al Sistema Embebido.
  9. Conectado el Sistema Embebido a la Plataforma de Hardware Virtual (Sección 1.2).
  10. FIN!!! Ver el comportamiento del hardware virtual en el programa.



1.5 - Explicación del programa “Hola Mundo!”

 

En este programa se realiza un llamado a todas las funciones de periféricos virtuales para demostrar de una sola vez todas sus funciones en acción. A grandes rasgos, el programa principal llama a la función HelloWorld() que es la que realiza las llamadas a las funciones del programa. Chequear que previo a entrar en el lazo infinito, el programa llama a vBoardInit(int baudrate); para inicializar la plataforma de hardware virtual a la velocidad deseada (esta velocidad debe coincidir con la velocidad seleccionada en el programa de la PC).

3 - Utilización de la biblioteca de Hardware Virtual

 

Para utilizar las funciones de hardware virtual no es necesario que únicamente se llamen a las funciones de hardware virtual. Es decir, el código puede mezclar tranquilamente código que acceda al hardware virtual y código que acceda al hardware real del Sistema Embebido. Ejemplos:

  • Se puede crear un código que sense un pin analógico real del Sistema Embebido y muestre ese valor en el display LCD del hardware virtual.
  • Se puede crear un código que al presionar un botón en el hardware virtual encienda un LED en el hardware real.


Las combinaciones se pueden realizar de cualquier forma y no hay ninguna restricción para mezclar el hardware real con el virtual.

 

3.1 - Selección de la plataforma embebida

 

En el archivo virtual_harware.h, en la sección macros (a partir de la línea 16) se encuentran las definiciones de las plataformas embebidas soportadas. Las posibles plataformas pueden ser:

 

 

  • BOARD_EDU_CIAA_NXP
  • BOARD_CIAA_ZERO
  • BOARD_PIC_CSS
  • BOARD_ARDUINO

 

 

Por defecto todas están comentadas y es una condición necesario que se descomente una (solo una) de las plataformas antes de realizar la compilación.

 

3.2 - Mapa de periféricos virtuales

 

La biblioteca embebida cuenta con una serie de periféricos virtuales (declarados en virtual_hardware.h). Los mismos son declarados en un enum del Lenguaje C. Prestar atención únicamente a los nombres de los periféricos, sus valores no tienen importancia dentro del Sistema Embebido; sirven para que el programa Virtual Hardware Platform los pueda identificar.

 

Todos tienen el prefijo ‘V_’, que significa virtual.

 

/**
 * Mapa de perifericos virtuales a los que se puede acceder.
 */
typedef enum VirtualPeriphericalMap {
    // Valores correspondientes a los leds
    V_LEDR = 'a',  
    V_LEDG = 'b',  
    V_LEDB = 'z',  
    V_LED1 = 'c',  
    V_LED2 = 'd',  
    V_LED3 = 'e',  
    V_LED4 = 'f',  
    // Valores correspondientes a las teclas
    V_TEC1 = 'g',
   V_TEC2 = 'h',
   V_TEC3 = 'i',  
    V_TEC4 = 'j',  
    // Valores correspondientes a los pines ADC
    V_ADC_CH1 = 'k'
    // Valores correspondientes a los pines DAC
    V_DAC_CH1 = 'n',
    // Valores correspondientes al periferico LCD
    V_LCD1 = 'o',  
    // Valores correspondientes al periferico 7 segmentos
    V_7SEG = 'p',  
} VirtualPeriphericalMap_t;




3.3 - Prototipos de funciones de hardware virtual

 

Los prototipos de las funciones soportadas por la plataforma de hardware virtual están declarados en virtual_harware.h. Los mismos son:



// Configura la plataforma

bool_t   vBoardConfig    (uint32_t baudRate);

// Funciones para utilizar los GPIO

bool_t   vGpioRead       (VirtualPeriphericalMap_t bluetoothPin);
void     vGpioWrite      (VirtualPeriphericalMap_t bluetoothPin, bool_t pinState);
void     vGpioToggle     (VirtualPeriphericalMap_t bluetoothPin);

// Funciones para el Display LCD
void     vLcdWriteString (VirtualPeriphericalMap_t display, LcdLine_t lcdLine, char * stringToWrite);

// Funciones para el ADC y DAC
uint16_t vAdcRead        (VirtualPeriphericalMap_t adcChannel);
void     vDacWrite       (VirtualPeriphericalMap_t dacChannel, uint16_t dacValue);

// Funciones para el Display 7 Segmentos
void     v7SegmentsWrite (VirtualPeriphericalMap_t display, uint8_t valueToShow);
